# Anthropic Faces User Backlash Over Claude Performance

**Date:** 2026-04-15
**Run:** 1
**Relevance Score:** 7/10
**Tags:** content, infrastructure

## Summary

Anthropic is facing significant user backlash over **perceived performance degradation** in Claude Opus 4.6 and Claude Code. Power users and developers accuse Anthropic of "nerfing" their models.

## Key Incidents

- **Stella Laurenzo complaint (April 2):** AMD Senior Director filed detailed GitHub issue claiming 67% performance drop
- **Claude Code head response:** Boris Cherny acknowledged issues, attributed some to UI-only changes ("redact-thinking-2026-02-12" header)
- **Outages:** Elevated timeout rates March 31-April 1; renewed issues April 8
- **Fortune coverage (April 14):** Major article on the backlash

## User Complaints

- "Claude Code is unusable for complex engineering tasks with the Feb updates" (#42796)
- "Claude Code's prediction-first behavior is dangerous on capital-at-risk projects" (#46212)
- Concerns about transparency around model changes

## Why This Matters

1. **Trust erosion:** Even loyal Claude users are questioning reliability
2. **Opportunity for alternatives:** OpenClaw's multi-model approach suddenly more attractive
3. **Content angle:** Users want transparency - this is a values story, not just tech
