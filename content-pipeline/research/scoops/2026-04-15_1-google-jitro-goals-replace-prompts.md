# Google Jitro: The Outcome-Driven Coding Agent

**Date:** 2026-04-15
**Run:** 1
**Relevance Score:** 8/10
**Tags:** infrastructure, content

## Summary

Google is developing **Project Jitro** (internally), their next-generation coding agent that evolves Jules from a task-executor to an **outcome-driven collaborator**. Expected to debut at **Google I/O on May 19, 2026**.

## Paradigm Shift

Instead of: "Refactor this function"
Now: "Improve performance by 15%" or "Increase test coverage to 80%"

The agent autonomously identifies what needs to change to hit a metric, rather than waiting for specific instructions.

## Key Differences from Current Agents

- **KPI-driven:** Set goals, not tasks
- **Repository-wide awareness:** Operates across entire codebases
- **Autonomous planning:** Determines what changes are needed to achieve outcomes
- **Self-iteration:** Continues until objective met

## Timeline

- Reported April 8, 2026
- Expected showcase: Google I/O (May 19, 2026)
- Evolution of Jules (current Google coding agent)

## Why This Matters

This is a genuine paradigm shift - from imperative ("do this") to declarative ("achieve this"). If it works, it fundamentally changes how developers interact with coding agents. Could pressure Claude Code and Cursor to follow.
