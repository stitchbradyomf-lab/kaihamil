# Content Pipeline

Organized content workflow for Kai Hamil.

## Folder Structure

```
content-pipeline/
├── drafts/           # Working documents not yet published
├── published/        # Final versions ready for deployment (empty = all deployed)
├── market-research/  # Research notes, problem mapping, market analysis
├── agent-docs/       # Architecture docs, org charts, operational reports
├── assets/           # Images, infographics, media files
├── scripts/          # Video scripts, audio scripts
└── archive/          # Published content (source .md files after HTML conversion)
```

## Workflow

1. **New content** → `drafts/`
2. **Ready for site** → `published/`
3. **After deployment** → source moves to `archive/`, HTML lives on site
4. **Research** → `market-research/` (may become content or stay reference)
5. **Operations** → `agent-docs/` (internal use)
