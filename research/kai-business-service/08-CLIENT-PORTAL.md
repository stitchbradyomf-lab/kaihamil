## Portal Overview

### URL Structure
- **Standard:** `clientname.kaihamil.com`
- **Custom domain:** `ops.clientbusiness.com` (Scale tier)
- **Fallback:** `portal.clearpath.io/clientname`

### Authentication
- Email/password with 2FA option
- Magic link login available
- Session timeout: 24 hours (configurable)
- "Remember this device" option

## Technical Implementation

### Hosting Options

**Option A: Subdomain Routing (Recommended)**
- Single deployment, multi-tenant frontend
- `*.kaihamil.com` wildcard SSL
- Client subdomain routes to their isolated backend
- Cost-effective, easier to maintain

**Option B: Separate Deployments**
- Each client gets own portal instance
- Custom domain support built-in
- More isolation, higher cost
- Better for Scale tier / enterprise

### Tech Stack (Suggested)

```
Frontend:        Next.js or SvelteKit (static-exportable)
Auth:            Clerk, Auth0, or custom JWT
Backend API:     OpenClaw instance (already running)
Database:        SQLite per client (on their VPS)
Hosting:         Vercel/Cloudflare (frontend) + Client VPS (API)
SSL:             Let's Encrypt / Cloudflare
```

### Architecture

```
┌─────────────────────────────────────────────────────┐
│                    CLOUDFLARE                        │
│              (DNS + SSL + CDN)                       │
└─────────────────────┬───────────────────────────────┘
                      │
          ┌───────────┴───────────┐
          ▼                       ▼
┌─────────────────┐     ┌─────────────────┐
│  PORTAL FRONTEND │     │  PORTAL FRONTEND │
│  (Vercel/CF)     │     │  (Vercel/CF)     │
│  sarah.kaihamil  │     │  john.kaihamil   │
└────────┬────────┘     └────────┬────────┘
         │                       │
         ▼                       ▼
┌─────────────────┐     ┌─────────────────┐
│  CLIENT VPS      │     │  CLIENT VPS      │
│  (OpenClaw API)  │     │  (OpenClaw API)  │
│  Sarah's data    │     │  John's data     │
└─────────────────┘     └─────────────────┘
```

### Security Model

| Layer | Implementation |
|-------|----------------|
| **Transport** | TLS 1.3, HTTPS only |
| **Authentication** | JWT tokens, optional 2FA |
| **Authorization** | Client can only access own data |
| **Data isolation** | Separate VPS per client |
| **Session** | Secure cookies, 24hr expiry |
| **Audit** | Full access logging |
| **Backups** | Daily automated, 30-day retention |

### API Endpoints (Client VPS)

```
GET  /api/status          → Agent health status
GET  /api/activity        → Recent agent actions
GET  /api/tasks           → Tasks and reminders
GET  /api/files           → File browser
GET  /api/files/search    → Search files
GET  /api/knowledge       → Knowledge graph summary
GET  /api/metrics         → Usage metrics
GET  /api/automations     → Cron job history
POST /api/tasks           → Create task
POST /api/chat            → Send message to agent
GET  /api/settings        → User preferences
PUT  /api/settings        → Update preferences
```

## Client Value Proposition

### "Check your system from anywhere"
> Open your laptop, see exactly what your AI has been doing. Files organized, tasks tracked, nothing hidden.

### "Visibility into what's happening"
> Unlike a black-box tool, you see every action. Morning brief sent? Check. Documents filed? See exactly where.

### "Audit trail of agent work"
> Every action logged. If you ever wonder "did I follow up with John?" — it's all there.

### "Peace of mind"
> Green status light = everything working. Red = we're already on it. Sleep better knowing someone's watching.
